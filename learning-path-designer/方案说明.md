# 游戏行业 AI Native 新人入职陪跑平台方案说明

---

## 一、问题诊断

游戏行业 AI Native 转型中，AI 工具已嵌入各岗位。传统培训以"制度宣讲+课程"为主，存在四大痛点：

1. **学用脱节**：员工学完仍不会用 AI 完成真实任务，融入周期 6 个月。
2. **缺乏个性化**：所有新人走同一路径，忽略岗位差异。
3. **能力无法量化**：HR/导师靠感觉判断，转正评估主观。
4. **知识不沉淀**：新人工作流无法沉淀为组织资产。

**本质**：传统 LMS 是课程交付，AI Native 需任务陪跑。

---

## 二、方案设计

### 产品定位

**学习路径设计器**——游戏行业 AI Native 组织新人融入 B2B 工具，助新员工 3 个月内从使用者转为协同交付者。

### 六大模块

| 模块 | 功能 | 价值 |
|------|------|------|
| Dashboard | 入职天数、AI 雷达图、导师反馈 | 新人看清状态 |
| AI 任务教练 | 输入任务→AI 拆解→Prompt→工具推荐 | "学"变"用" |
| 学习路径 | 3 阶段 12 周路径，动态推进 | 替代人工计划 |
| 岗位工作流库 | 5 岗位 AI 工作流模板 | 快速理解岗位 AI |
| HR/导师看板 | 进度、风险预警、交付评估 | 数据化管理 |
| 成长报告 | 学习轨迹、能力变化、转正建议 | 转正评估依据 |

---

## 三、AI 工具选型理由

| 工具 | 理由 |
|------|------|
| React 18+TS | 类型安全，组件化适合多页面，生态成熟 |
| Vite 5 | 毫秒级 HMR，ESM 构建快 10-20 倍 |
| Tailwind CSS 3 | 设计 Token 保一致性，避免 CSS 膨胀 |
| Framer Motion 11 | 声明式动画，layout 属性自动处理过渡 |
| Recharts | 纯 React 组件，雷达图直观，比 ECharts 轻量 |
| 腾讯云 CloudBase | 静态托管零成本，上海节点快，可扩微信生态 |


### 路由（HashRouter）
CloudBase 不支持服务端路由重写，用 HashRouter 避免刷新 404。

| 路由 | 页面 | 权限 |
|------|------|------|
| `#/` | Dashboard | 员工/HR |
| `#/coach` | AI 教练 | 员工 |
| `#/path` | 学习路径 | 员工/HR |
| `#/workflow` | 工作流库 | 员工/HR |
| `#/mentor` | 导师看板 | HR 仅 |
| `#/report` | 成长报告 | HR 仅 |

### 角色权限
- **员工**：Dashboard/教练/路径/工作流/报告(自)
- **HR**：全部页面，可查看所有新人进度

角色通过 `AppContext` 管理，localStorage 持久化。

### 视觉规范
- 背景 `#F8FAFC`，卡片 `#FFFFFF` 微阴影
- 主色 `#4F46E5` 靛蓝，强调色 `#06B6D4` 青绿
- 字体：Lexend(标题)+Source Sans 3(正文)

## 六、总结

本方案针对游戏行业 AI Native 组织新人融入痛点，设计"任务陪跑"B2B 工具。六大模块实现"学"到"用"闭环。当前已完成 MVP 开发和设计重构，下一步完善 AI 功能。
